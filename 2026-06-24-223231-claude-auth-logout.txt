● Step 7 — systemd/mural-core.service: Done.

  Key decisions and why:

  - After=plasma-plasmashell.service + Wants= — After= alone is safe on non-KDE
  systems (systemd silently ignores missing unit names in ordering constraints).
  Wants= starts plasmashell if not running but doesn't fail if it doesn't
  exist. This is the DEVGUIDE's solution to hard problem #3.
  - Environment=DBUS_SESSION_BUS_ADDRESS=unix:path=/run/user/%U/bus — %U is
  systemd's numeric UID specifier for user services; this is the fix for hard
  problem #4. Without it dasbus can't reach the session bus from a
  systemd-launched process.
  - PassEnvironment=DISPLAY WAYLAND_DISPLAY ... — lwe needs the display
  variables; these are inherited from the graphical session when the user runs
  systemctl --user import-environment (standard practice in
  /etc/environment.d/).
  - KillMode=control-group — ensures lwe (a child process) is killed when the
  service stops, the safety net for hard problem #5 on top of what BackendRunner
  already does.
  - TimeoutStopSec=8 — gives the BackendRunner SIGTERM→SIGKILL 3s window room
  before systemd escalates.
  - StartLimitBurst=5 / StartLimitIntervalSec=60 — prevents thrash-restarting if
  lwe is missing.
  - Slice=app.slice — correct placement in the systemd cgroup hierarchy for
  graphical user apps.
